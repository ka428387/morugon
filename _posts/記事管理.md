# 📋 記事管理ダッシュボード

## ✍️ status 運用ルール

記事の `status` は次の流れで進めます。front matter の `status:` を手で書き換えて運用します。

`draft`（下書き） → `editing`（校正中） → `ready`（公開待ち） → `published`（公開済み）

- **draft** … 書きかけ・素材段階
- **editing** … 自分で校正・推敲している最中
- **ready** … 校正OK、あとは公開日を待つだけ
- **published** … 公開済み（`_posts` に置かれ、`date` が到来したもの）

> ℹ️ このダッシュボードの表は **Dataview プラグイン**で表示されます。未インストールの場合はクエリがコードのまま表示されるので、コミュニティプラグインから「Dataview」を有効化してください。

---

## 📝 未公開の下書き一覧（published 以外）

```dataview
TABLE WITHOUT ID
  file.link AS "記事",
  status AS "状態",
  categories AS "カテゴリ",
  permalink AS "パーマリンク"
WHERE status AND status != "published"
SORT status ASC, date ASC
```

---

## ✅ 公開準備OK一覧（status: ready）

```dataview
TABLE WITHOUT ID
  file.link AS "記事",
  categories AS "カテゴリ",
  date AS "予定日",
  permalink AS "パーマリンク"
WHERE status = "ready"
SORT date ASC
```

---

## 📊 カテゴリ別 下書き本数（published 以外）

```dataview
TABLE WITHOUT ID
  カテゴリ,
  length(rows) AS "本数"
WHERE status AND status != "published"
FLATTEN categories AS カテゴリ
GROUP BY カテゴリ
SORT length(rows) DESC
```

## 📚 公開済み記事一覧（新しい順）

```dataview
TABLE WITHOUT ID

  permalink AS "パーマリンク"
```
