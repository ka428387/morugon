ブログ「morugon」（ka428387.github.io/morugon/）の記事下書きをAIと作成するための指示書。
**最重要は「2. 文体」**。ここを外すと全部やり直しになる。

## 1. ブログと書き手の基本情報

- ブログ名: morugon（もるごん）。テーマは「暮らしのキュレーション」
- カテゴリ: あそび／写真／育児／生活／思考／AI

- 記事は「1記事1エピソード」で小分けにする。長くなったら分割を提案する

## 2. 文体（最重要）

### 基本
- 敬体（です・ます）がベース。**感情が動くところ・言い切りたいところで平文を混ぜる**
  - 例:「どんどん、作りたくなってしまった。」「全然問題ない。」「目分量の3等分はけっこう難しい。」
- 体言止めの導入もよく使う。例:「先日4歳のお誕生日を迎えた次男。」「問題は、上の2人。」
- 絵文字は使わない。「！」は自画自賛や気持ちの高まりで使ってよい
  - 例:「きれいなグラデーション！そして美味しそうなケーキ！」
- 読者への呼びかけもたまに使う。例:「すごくないですか？」「ぜひ写真にもおさめてみてください」

### カッコ書きのルール
- カッコは**「演出のツッコミ」ではなく「事実の補足・本音」**に使う
  - ○ 良い例:（義母に事後報告のLINEはしました）（なぜ今までほとんど出したことのないピーマンの肉詰め？？）（割りました。）（私セレクト）
  - × 悪い例:（いつものやつです。）（ボンッて言いました。）（しんみり）——作られたツッコミ感が出るのでNG
- 二連カッコもあり。疑問はそのまま疑問として置く

### 禁止事項
- **「効く」という表現は使わない**（→「役に立つ」「助かる」「ありがたい」などに）
- **締めで名言・決め台詞を作らない**。「夜の機嫌は、朝の足で作られる」「モノは足し算、仕組みは掛け算」のようなコラム的キメはNG
- 毎段落を教訓に変換しない。観察は観察のまま置く。「〜なのだと思います」の連発をしない
- 出来事を圧縮しすぎない。時系列の実況・状況の具体（「お誕生日前日」「途方に暮れていた私は〜を思い出した」）を残す

### 締め方
- うまい回収より、**気持ちをぽんと置く／誰かへの呼びかけ**で終わる
  - 例:「楽しみに待っているよ。」「お気に入りの遊びです。」「真夏になる前に、おすすめの外遊びです。」
- 日記なので、ちゃんとした結論がなくてもいい（あってもいい）

### 感情の持っていき方
- 失敗や本音を先に正直に出す →そのあと良さを書く、の順が信頼される
  - 例:「もちろん、まったく騒がなかったわけではありません。でも〜」
- 自虐は軽く、明るく。「我ながら『やばい嫁だな』と思いました」
- 見せられないもの・できないことは、正直に言って魅力に転化する
  - 例:「しっかり顔が写っているので載せられないのが残念ですが、シャボン玉越しの子ども、とってもかわいいですよ」
- 元フォトグラファーとして「写真で確認した」証言が使える。「過去の写真を見返してみると、1歳4ヶ月の三男も〜」
- 小さな伏線→回収も好む。例: 購入リストに「（私セレクト）」→後段で「（ごめんラッパ……）」

### 参考にすべき公開記事（文体の正解）
- 「高すぎて何度もためらった。我が家がキュボロを迎えるまで」
- 「行き当たりばったりなお誕生日ケーキ」→ **全文を「7. 文体見本」に同梱。まずこれを読むこと**
- 「好きなものをたくさん撮ってほしい」
- プステフィックスのシャボン玉記事

## 3. 記事の体裁ルール

### front matter（雛形）
```yaml
---
layout: post-with-share
title: "タイトル"
date: YYYY-MM-DD        # 仮の日付でOK
image: /assets/images/TBD.jpg
categories: [あそび]     # 1つ
tags: [あそび]           # 複数可。AI関連なら [生活, AI活用] など
permalink: /entry/YYYYMMDD/
excerpt: "抜粋文。本文の魅力的な一文を使ってよい"
---
```

### 本文ルール
- 本文にH1は書かない（titleが見出しになる）。セクション見出しは `## `（半角スペース必須、末尾##不要）
- **本文の先頭に画像を置く**（公開記事はすべてこの形）
- 画像タグ: `<img src="{{ '/assets/images/TBD.jpg' | relative_url }}" alt="内容に即した説明" style="margin: 1.5em auto;">`
- リンクは内部・外部とも新しいタブ: `<a href="{{ '/entry/YYYYMMDD/' | relative_url }}" target="_blank" rel="noopener noreferrer">〜</a>`（外部はURL直書き）
- 太字 `**〜**` はここぞの一文に少しだけ
- アフィリエイト商品がある記事は、末尾にPR枠:
```html
<div style="margin-top: 2em; padding: 1em 1.2em; background-color: #f7f7f7; border-radius: 8px; font-size: 0.85em; color: #666;">
  <p style="margin: 0 0 0.6em 0;">記事で紹介した商品はこちら　<span style="opacity: 0.6;">[PR]</span></p>
  <p style="margin: 0;">・<a href="リンク" target="_blank" rel="noopener noreferrer">商品名（楽天市場）</a></p>
</div>
```

## 4. プライバシールール

- 子どもの名前は書かない（長男・次男・三男）。顔が写った写真は載せない
- 本名（フルネーム）・会社名・市町村名は書かない。「北関東の田舎」まで
- 家族・友人は「義妹」「友人」など関係性のみ。固有名は出さない
- 年齢は「出来事当時の年齢」で書く（プロフィールのみ現在基準）

## 5. 下書き作成のワークフロー

- 素材は、本人が提供したもの（Xポスト・日記・メモ・口頭説明）のみから書く
- **AIが推測で埋めた箇所は、納品時に必ず明示する**（本人が事実に直す前提）
- 本人が書き足すべき箇所は本文中に〔〕で示す。例:〔リンク〕〔子どもの反応をここに〕
- 下書きはたたき台。本人が自分の言葉で書き直して公開する、という往復が前提
- Git運用: 記事ごとに `draft/記事スラッグ` ブランチ → 完成後mainへマージで公開
- 相互リンクがある記事は同時公開（片方だけだとリンク切れ）
- レシピの分量はグラム単位、金額は円表記

```markdown
先日4歳のお誕生日を迎えた次男。どんなケーキが食べたい？と聞いたら、予想外の「ブルーベリーのケーキが食べたい」というリクエスト。

当初の構想はささやかなものでした。パウンドケーキを焼いて、生クリームとブルーベリーをトッピングとして添える程度でいいかな、と。ところがお誕生日前日、家にクリームチーズが2箱あることに気づいてしまい、じゃあチーズケーキにしようかな、と調べはじめたら——どんどん、作りたくなってしまった。


行き着いたのは、ブルーベリーの3種のムースを重ねるレアチーズケーキでした。参考にしたのは、<a href="https://cookpad.com/jp/recipes/20083359" target="_blank" rel="noopener noreferrer">クックパッドのこちらのレシピ</a>です。

<img src="{{ '/assets/images/blueberry-cake-01.jpg' | relative_url }}" alt="ムースを作っている調理中の様子" style="margin: 1.5em auto;">

ところがここで問題が発生します。ラウンド型のケーキ型が、家にない！

誕生日当日の朝、ケーキ型が売っていないかと何軒か見て回ったものの、見つかりません。どうしよう・・・と途方に暮れていた私は、以前義妹ちゃんがケーキを焼いていたのを思い出したのです！そんなわけで夫の実家に合鍵を使って勝手に入り、製菓用具の収納棚から勝手に拝借してきました。我ながら「やばい嫁だな」と思いました。（義母に事後報告のLINEはしました）

難しいかな？と思っていたケーキ作りですが作ってみると意外と大丈夫でした。バターをレンジで爆発させるというハプニングもありましたが、そんなのお掃除すれば大丈夫。全然問題ない。

「オーブンで焼く」という工程がないので、材料さえ揃えてしまえばあとはレシピに忠実に倣っていくだけです。いちばんの山場である3種類のムース作りまで、30分ほどでできました。その間、三男がひとりで遊んでいてくれたのが、何よりのアシストでした。

今回、地味に役に立ったのがAIです。参考にしたレシピの分量をClaudeに読み込ませて、すべてグラム単位に変換してもらいました。大さじや個数表記が消えて、計りに乗せていくだけになる。さらに、途中に「ムースを3等分にする」という工程があるのですが、そこまでの合計の分量から「何グラムずつにすればいいか」を計算してもらえたのが助かりました。目分量の3等分はけっこう難しい。

そうして、なんとか完成したのがこちらです。

<img src="{{ '/assets/images/blueberry-cake-02.jpg' | relative_url }}" alt="完成したブルーベリーのレアチーズケーキ" style="margin: 1.5em auto;">

行き当たりばったりのわりに、きれいなグラデーション！そして美味しそうなケーキ！

夜は義母も誘って、みんなでごはん（これまた次男リクエストのピーマンの肉詰め）（なぜ今までほとんど出したことのないピーマンの肉詰め？？）を食べて、ケーキを食べました。実は義母と次男は誕生日が同じ日なのです。だからこの日は、ふたりの合同お誕生日会。一緒におめでとうを言い合える相手がいるのは、次男にとって特別なことなのかもしれません。

ブルーベリーケーキのリクエストに全力で応えた母の記録として、次回のための自分用メモを残しておきます。

- 下地のクッキーはなるべく細かく潰す
- ムースの3等分は、上にかけるソースの分量を考慮して140g／160g／180gくらいの傾斜配分にしてみる
- 白いソースは温める可能性があるので、レンジ可の容器に入れる
- トッピング用のジャムは多めに残しておく

次男のリクエストをきっかけにまたひとつ製菓レベルが上がってしまった。来年のお誕生日にはどんなリクエストがくるかな。楽しみに待っているよ。
```
